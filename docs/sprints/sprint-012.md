# Sprint 012 — G1 仕上げ: 週次ふりかえりと索引運用

- Type: main
- 主眼: 日々の原本から週次を振り返り、記憶が増え続けても索引を安全に保てる運用を完成させる。dashboardとmigrationは承認済みの条件に従い、実装前に明示判断する。
- 依存: sprint-011 done。三層記憶、timeline、settings、既定値＋上書きが全て回帰で保護されていること。

## 外から見える成果

1. 「今週を振り返って」で、対象週の日次journal原本から、活動・決定・翌週への申し送りを区別した振り返りを得られる。
2. `MEMORY.md` が200行へ近づくと、古い月の退避候補と影響が提示され、ユーザー確認後だけ整理できる。
3. dashboardと既存ユーザーmigrationを実装するかしないかが、条件と根拠つきで明示される。

## スコープ

### A. weekly

- 週次ふりかえりは毎回、対象期間の日次journal原本を読む。前週要約から次週要約を作る「要約の要約」をしない。
- activities、decisions、nextを区別し、topicや外部根拠を使う場合は参照元を示す。
- 矛盾する決定は自動統合せず、新しい決定を優先表示したうえで、統合候補をユーザーへ確認する。
- 出力をローカル成果物に保存する場合は既存規約とシームを通す。

### B. 索引退避運用

- `MEMORY.md` の200行上限、journal月単位索引、topics索引を維持する。
- 上限接近／超過時はexit 0＋stderr警告から、古い月の退避候補、残る参照、予想されるtimelineへの影響を提示する。
- 退避・削除・索引更新は対象提示→明示確認の2段階。自動で消さない。
- 退避後もtimelineやweeklyの対象期間を扱う方法を案内し、記録を見失わせない。

### C. dashboard 判断ゲート

- G1の達成条件はtimelineであり、dashboardは必須ではない。
- sprint-012開始時点の利用反応を確認し、実装する／見送るをユーザーが判断できる短い材料を出す。
- 実装の明示判断がある場合だけ、このsprintのscope changeとして契約をPlannerが更新してから着手する。無断実装しない。
- 見送りの場合は、その判断と理由がprogressに残れば本sprintの不合格理由にしない。

### D. migration 判断ゲート

- 第2期配布が新規セットアップのみならmigrationは作らない。
- 既存ユーザーがいると確認された場合だけ、journal/topicsディレクトリの安全な追加とpreferences v1→v2 migrationを、このsprintのPlanner更新後に実装する。
- 既存記憶のバックアップ、dry-run、空上書き禁止、手書き行保持、ロールバック手順を受入条件へ追加してから着手する。

## スコープ外

- restoreシーム「昨日の状態に戻して」。
- 自動削除・無確認のdecision統合。
- dashboard／migrationの無断追加。
- 外部データ本文の保存。

## 受入基準

1. **原本起点（C3/C5）**: 週次入力が各日journal原本であることを実行ログで示し、過去の週次要約だけから生成していない。
2. **週境界（C3）**: `CC_SECRETARY_NOW` と固定fixtureで週のfrom/to、日付跨ぎ、データ0件を決定的に処理する。
3. **分類（C1/C3）**: did/decided/nextが区別され、決定原文と変更履歴を勝手に要約・統合しない。
4. **外部根拠（C5）**: 外部事実にはサービス名＋URL/ID＋日付があり、本文複製がない。
5. **保存（C3/C5）**: 保存時だけ成果物シームを通り、journal副作用と日本語commitが1回ずつ発生する。
6. **200行運用（C3/C5）**: 199/200/201行相当のfixtureで、索引維持、exit 0＋stderr警告、候補提示、無確認変更0件を確認する。
7. **退避2段階（C5/C7）**: 対象と影響を先に示し、明示確認後だけ実行する。キャンセル時の副作用0件。
8. **dashboard判断（C1）**: 実装／見送りの判断と根拠が記録される。実装する場合はPlannerが契約を更新済みで、追加受入基準を満たす。
9. **migration判断（C1/C5）**: 既存ユーザー有無と実装／見送りが記録される。実装する場合はPlannerが安全なmigration基準を追記済み。
10. **週次模擬会話（C4/C7）**: 既定3行で結果と次の一手を伝え、提案は1つまで、無断整理しない。明示「くわしく」時だけ補足1つ。
11. **無回帰（C6）**: sprint-011までの全回帰＋weekly/索引運用の新規assertが0 FAIL。

## 評価証跡

- 固定週fixtureとweekly入力元一覧、生成結果。
- 199/200/201行のreindex・警告・確認／キャンセル結果。
- dashboardとmigrationの判断記録。実装した場合は更新済み契約と追加証跡。
- 全回帰のPASS/FAIL集計。
