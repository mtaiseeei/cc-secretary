# Sprint 002 — 記憶ケアと自動コミット

- Phase: P1（秘書コア）
- 主眼: 秘書の「記憶」を蓄積・保護し、中断から復元できるようにする。作業の節目でローカル自動コミットする。
- 依存: sprint-001（ワークスペースとルーターが存在すること）

## スコープ（含む）

- `skills/memory-care/` の実装（F05）。
  - 記憶の追加・更新（`memory/decisions/`、`memory/preferences.md`）。
  - 保護規則: 空内容で上書きしない／削除前に日常語で警告・確認／`MEMORY.md` 索引を常に最新に。
  - 再起動しおり（`_resume.md`）プロトコル: 中断時に文脈を書き出し、再開時に読み込む。
  - オンデマンド振り返り（「振り返りして」）: 進行中案件・決定を棚卸し（自動 `case-NNN` 生成はしない）。
- 自動コミット（F07）: 作業の節目で `secretary/` 内をローカル commit（日本語メッセージ）。push はしない。

## スコープ外

- コネクタ参照（daily / setup-google）は sprint-003。

## 受入基準（薄い契約・詳細化は着手時）

- 記憶の追加・更新・索引反映がドライランで検証できる。
- 空上書き拒否・削除前警告が発火することをスクリプトで実証できる。
- `_resume.md` の書き出し→読み込みで文脈が復元される。
- 節目で日本語メッセージのローカルコミットが作られ、push は行われない。
