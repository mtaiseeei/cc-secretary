# Sprint 002 — 記憶ケアと自動コミット

- Phase: P1（秘書コア）
- 主眼: 秘書の「記憶」を蓄積・**保護**し、中断から復元できるようにする。作業の節目で日本語メッセージのローカル自動コミットを行う（**push はしない**）。
- 依存: sprint-001（ワークスペース・ルーター・テンプレート）＋ sprint-001-patch-001（`${CLAUDE_PLUGIN_ROOT}` 参照・`plugins/cc-secretary/templates/` 配置が確定済み）

## なぜこのスプリントか

sprint-001 は「秘書の家ができて初回コミットまで通る」ところまで。sprint-002 は、その家に住む秘書が**記憶を安全に育て、中断しても文脈を失わない**状態を作る。非エンジニアにとっての価値は「言ったことを覚えていてくれる・勝手に消さない・続きから再開できる」という信頼。daily／コネクタは sprint-003 に分離する。

## スコープ（含む）

### 1. memory-care スキル（F05）
- `plugins/cc-secretary/skills/memory-care/SKILL.md`（frontmatter 有効・`name` 一意・薄いルーターから段階ロード）。
- 扱う導線（すべて `secretary/` 配下のフラット構造。`.company/` 等の隠し・部署構造は作らない）:
  - **記憶の追加・更新**: `memory/decisions/YYYY-MM-DD-decisions.md`（決定ログ）、`memory/preferences.md`（好み・口調・環境）。相対日付は絶対日付に変換して記録。
  - **保護規則**（不変条件・DESIGN.md／constraints 由来）:
    - 空内容・実質空（空白のみ）で既存記憶を**上書きしない**。
    - 削除の前に「何を消すか」を**日常語で警告**し、確認を取ってからのみ実行。
    - `memory/MEMORY.md` 索引（1行1記憶）を、記憶ファイルの増減に**常に追従**させる。
  - **再起動しおり**: `secretary/memory/_resume.md`（または DESIGN.md の `_resume.md` プロトコル準拠の場所）に、中断時の文脈（進行中案件・次アクション・未確定事項）を書き出し、再開時に読み込んで文脈を復元する。ルーター起動時に `_resume.md` の有無をチェックし、あれば「前回の続き」を提案する。
  - **オンデマンド振り返り**: 「振り返りして」等の自然な言い回しで起動。進行中案件・決定を棚卸しして記憶を整理する。**自動 `case-NNN` 生成・`patterns/` 自動統合はしない**（オンデマンドのみ）。

### 2. 自動コミット（F07）
- 作業の節目（本スプリントでは主に**記憶更新**の節目）で、`secretary/` 内をローカル commit（**日本語メッセージ**、何をしたかがわかる粒度）。
- **push はしない**（ユーザーの明示指示時のみ。リモート設定・push を既定導線に含めない）。
- sprint-001 の「オンボーディング完了時の初回コミット」を、汎用の「節目コミット手順」に一般化する。成果物・案件区切りの節目は sprint-003 以降で同手順を再利用する。

### 3. 検証シーム（対話 Claude に依存しないドライランのため）
- 上記の**決定的な部分**（索引の追従、空上書き拒否、削除前ガード、`_resume.md` の書き出し/読み戻し、節目コミットの生成と push 抑止）は、スクリプトから検証できる**決定的なシーム**（ヘルパー手順／スクリプト等）として実装する。LLM 判断に委ねる自然言語の入口は別途置いてよいが、**保護規則の発火はスクリプトで assert 可能**であること。
- 実装手段（ヘルパーの言語・粒度）は Generator の裁量。ただし決定性（同じ入力→同じ結果）を満たすこと。

### 4. ルーターへの組み込み
- `skills/secretary/SKILL.md` に、記憶ケア（覚えて／記憶／削除／振り返り／前回の続き 等の言い回し）のモード判定を追加し、`${CLAUDE_PLUGIN_ROOT}` 相対で memory-care を段階ロードする。起動時 `_resume.md` チェックもここに接続。

## スコープ外（このスプリントでやらない）

- 今日やること（daily）・コネクタ参照・出力規約の実運用 → sprint-003
- 成果物作成・案件区切りの節目コミット運用 → sprint-003 以降（手順は本スプリントで一般化するが、発火導線は各機能スプリントで接続）
- setup-google / microsoft / notion → sprint-003, 004
- やさしいハーネス・build → sprint-005

## 受入基準（この契約は厚めに定義する）

Evaluator は `docs/spec/rubric.md` の方法で以下を検証し、証跡（実行コマンドと結果）を `docs/feedback/sprint-002.md` に残す。

1. **スキル構文（C2, ゼロ許容）**: `memory-care/SKILL.md` の frontmatter が有効・`name` 一意。ルーターからの参照が `${CLAUDE_PLUGIN_ROOT}` 相対で、デッドリンクがない。
2. **索引の追従（C1, C3）**: 記憶ファイルを追加/削除するドライランで、`MEMORY.md` 索引が1行1記憶で追従する（追加で行が増え、削除で該当行が消える）ことを assert。
3. **空上書き拒否（C3, ゼロ許容の不変条件 C5）**: 既存記憶に対する空・空白のみの上書きを試みると、ガードが**拒否**して既存内容が保持される（スクリプトで exit 非ゼロ／内容不変を assert）。
4. **削除前警告（C3, C5）**: 削除導線が、確認なしでは実行されず「何を消すか」を提示して中断する（確認フラグ相当がない限り削除されないことを assert）。
5. **再起動しおり（C1, C3）**: `_resume.md` の書き出し→読み戻しで、記録した文脈項目が復元される。ルーター起動チェックが `_resume.md` 有無を検知する。
6. **自動コミット（C1, C3）＋ push 禁止（C5, ゼロ許容）**: 記憶更新の節目で `secretary/` に**日本語メッセージのコミットが1件以上**作られ、**リモート設定・push が発生していない**（`git log` に日本語メッセージ、`git remote` 空、push 実行なしを assert）。
7. **非エンジニア体験（C4）**: 削除警告・しおり提案・振り返りの出力が日常語＋具体例で、報告が3行型（「次に何が起きるか」を含む）。`rules/plain-language.md` を参照。
8. **安全・規律（C5, ゼロ許容）**: `~/workspace/agentic-harness` に書き込みなし。外部データ同期層を作らない。資格情報を記憶に書き込まない/コミットしない。単段クレジット方針・6規律に反する記述がない。
9. **無回帰（C6, ゼロ許容）**: sprint-001＋patch-001 の回帰スイート（既存 assert）が全パス。本スプリントの assert を追加し、実行コマンドを progress に記録。

### rubric 対応まとめ
- C1 完成度: 2,5,6 / C2 構文: 1 / C3 機能実証: 2,3,4,5,6 / C4 体験: 7 / C5 安全・規律: 3,4,6,8 / C6 無回帰: 9

## Generator への引き継ぎメモ

- **参考にしてよい company の一次情報**（`~/workspace/inbox/company`、MIT・クレジット継承、**読むだけ**）:
  - MEMORY.md 索引ルール: SKILL.md 867-872 行
  - 空上書き防止・削除警告: SKILL.md 914-935 行
  - 再起動しおり `_resume.md` プロトコル: SKILL.md 647-661 行、起動時チェックは 248-262 行
- **cc-secretary 流への読み替え（重要）**:
  - company の部署制・キーワード振り分け・`.company/` 構造・部署間 inbox 通知の名残を**持ち込まない**。記憶は `secretary/memory/` の**フラット構造**（`decisions/` と `preferences.md` と `MEMORY.md`）に限定。
  - 自動 `case-NNN` 生成・`patterns/` 自動統合は**廃止**（振り返りはオンデマンドのみ）。
  - パス参照は `${CLAUDE_PLUGIN_ROOT}` 相対（patch-001 で確定）。リポジトリ直下相対・絶対直書きを使わない。
- **company にない新規実装**: 節目ごとの**日本語ローカルコミット手順**（company は git 運用しない設計）。push を既定導線に含めないこと。
- 回帰スクリプトは、一時ディレクトリに `secretary/` を用意し、記憶操作→索引/保護/しおり/コミットを assert する形が扱いやすい。
