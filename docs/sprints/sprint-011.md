# Sprint 011 — G2 settings と preferences v2

- Type: main
- 主眼: 既定規律を維持したまま、役割・言葉遣い・詳しさ・確認方法を明示設定で変えられる「100人100通りの秘書」を実現する。
- 依存: sprint-010 done。設定変更の記録先となるjournalと節目体験が成立していること。
- **先行ゲート**: constraints/rubric/憲章テンプレ/回帰を「既定値＋opt-in上書き」へ揃え、その検査が通るまで settings本体の実装に入らない。

## 外から見える成果

1. 初回は5問以内で、仕事・役割と説明の詳しさを含めて設定できる。口調は聞かず標準で開始し、後から変更できると分かる。
2. 「もっとフランクに」「専門用語そのままで」「呼び方を変えて」等でいつでも設定を変えられる。
3. 変更前に例文を確認でき、変更後に覚えた内容が明示され、journalとローカルcommitへ残る。
4. 役割が提案・例示・用語補足へ実際に反映される。
5. 設定ファイルが無い・空・一部欠損でも安全な既定値で動く。

## 実行順序

### Gate A — 先行規約を先に完了する

1. `docs/spec/constraints.md` と `docs/spec/rubric.md` が、既定3行＋明示時だけ補足、一般技術用語維持、既定値採点を正本化していることを確認する。Generatorはこれらを編集しない。
2. 配布する `templates/AGENTS.md` と `templates/CLAUDE.md` の絶対規約を、**既定3行・preferencesで「くわしく」が明示された場合だけ3行＋補足1つ**へ改訂する。
3. `rules/plain-language.md` を、全員共通で不変の第1部と、preferencesを読む第2部へ分ける。
4. regression section 5/9を、固定3行や固定語彙の単純検査ではなく、既定値＋許可された上書き＋禁止事項を検査する形へ更新する。
5. Gate Aの回帰が0 FAILになるまで、以下のsettings実装を開始しない。progressにGate A完了の証跡を残す。

### Gate B — settings本体

- 初回と途中変更を1つの `settings` スキルで扱う。
- 初回質問は5問以内。既存3項目＋仕事・役割＋説明の詳しさ。口調は質問しない。
- 途中変更は自然な発話から入り、例文プレビュー→1行確認→指定項目の反映→「こう覚えました」→journal `did`→節目commitの順に進む。
- `memory-tools.sh pref-set <セクション> <キー> <値>` は指定行だけを変更し、他の手書き内容を保持する。
- `memory-tools.sh pref-note-add <本文>` は秘書のメモへ追記する。自発提案からの追記も1行確認を先に行う。
- categoricalな挙動スイッチを自由メモに埋めない。

### Gate C — preferences v2 と適用

- 基本: 呼び方／仕事・役割／主に使うサービス。
- 言葉遣い: 口調、専門用語、報告の詳しさ、決定の確認。
- 口調のお手本: NG/OK例ペア。
- 秘書のメモ: 確認済みの自由追記。
- 既定: 丁寧（標準）、専門用語=ふつう、報告=みじかく、決定確認=都度。
- `standard` / `friendly` / `formal` の3プリセットを提供する。濃いキャラクターは同梱しない。
- 全スキルの必要箇所と、別repo `yasashii-harness` のagents overlayは plain-language と preferences の両方を読む。
- 役割に応じ、営業→商談メモ、講師→講義資料、経営→数字のまとめ等、提案・例示・補足の題材を変える。
- `決定の確認: まとめて` の場合はsprint-010の締めプロトコルへ接続する。

## スコープ外

- 関西弁・執事風等の濃いキャラクタープリセット。
- output stylesへの依存。
- 個人化された文面の完全一致を固定する回帰assert。
- preferences v1→v2 migration（既存ユーザーがいる場合のみsprint-012で判断）。

## 受入基準

1. **先行ゲート順守（C1/C5）**: templates、plain-language、section 5/9の変更と0 FAIL証跡が、settings本体着手前の作業順としてprogressに記録される。
2. **既定＋上書き（C2/C4）**: preferences無し・空・部分欠損では既定3行。明示「くわしく」だけ3行＋補足1つ。他設定が報告長を暗黙変更しない。
3. **一般技術用語（C4/C7）**: 「ことば添え」でも一般語を置換せず、対象語だけ短く補足する。「そのままOK」でも安全説明を省かない。
4. **初回5問（C1/C4）**: 仕事・役割と詳しさを含む5問以内。口調は聞かず、後から変更できる案内がある。
5. **更新シーム（C3/C5）**: `pref-set` が対象行だけを変え、他項目・手書き行を保持する。`pref-note-add` は追記のみ。空・未知key・境界外を拒否する。
6. **確認フロー（C3/C7）**: 例文→確認→反映→宣言→journal→commitの順を実動作で確認する。キャンセル時はファイル・journal・commitに副作用0件。
7. **役割の写像（C3/C7）**: 営業・講師・経営の少なくとも3役で、同じ依頼への提案／例示が役割に寄る。設定にない事実を捏造しない。
8. **3設定模擬会話（C3/C4/C7）**: 既定、フランク＋そのままOK、きっちり敬語＋ことば添え＋くわしくで同一タスクを実行し、許可範囲だけ変化する。
9. **決定確認切替（C3）**: 都度とまとめての両方がsprint-010の節目プロトコルへ正しく接続し、未確認記録・拾い漏れ確認の省略がない。
10. **persona drift防止（C3）**: 新セッション相当でpreferencesを再読し、設定を維持する。output style無しでも成立する。
11. **無回帰（C6）**: sprint-010までの全回帰＋Gate A/B/Cの新規assertが0 FAIL。C2/C5/C6の閾値をやさしさで相殺しない。

## 評価証跡

- Gate Aの差分、section 5/9の意図的失敗検出、全回帰。
- 初回5問と途中変更の模擬会話。
- pref-set/pref-note-add前後のファイル差分、キャンセル時副作用。
- 3設定×同一タスク、3役割×同一依頼の比較。
- 新セッション相当の再読結果。
