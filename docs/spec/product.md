# Product

## これは何か

`cc-secretary` は、**非エンジニア向けのAI秘書プラグイン**（Claude Code plugin / public / MIT）。
「ゆるAIコーディング塾」第2期以降の目玉コンテンツとして配布する。

設計の中心思想は **「データは外に置いたまま、秘書だけがローカルに住む」**。
外部データ（メール・予定・ファイル・タスク）は各SaaSに置いたまま公式リモートコネクタで都度参照し、
秘書の記憶（決定・好み・進行中案件）と成果物（文書の正本）だけをローカルに置いて、節目ごとに自動コミットする。

正本の設計方針は `docs/DESIGN.md`。この spec 群はそれを実装可能な単位に落としたもの。

## 対象ユーザー

- **主対象（エンドユーザー）**: ゆるAIコーディング塾の受講者。非エンジニア30〜60代。
  - 標準環境は Claude デスクトップアプリ／Claude Code。第1回座学で Git / GitHub を習得済み。
  - 専門用語・英語のエラー文・技術的な段取りに不慣れ。「今何が起きているか」が見えないと不安になる。
- **副対象（配布・保守者）**: 村山さん（塾運営者）。プラグインを保守し、カリキュラムに組み込む。

## ゴール

1. **3コマンドで入る**: cc-company と同じインストール導線（marketplace add → install → `/secretary`）を維持する。
2. **初回で秘書ディレクトリができる**: オンボーディングが数問のやさしい質問だけで、秘書のワークスペース（`secretary/`）を生成し `git init` する。
3. **記憶が守られる**: 決定・好み・進行中案件がローカルに蓄積され、空上書き・誤削除から保護される。再起動しおりで文脈が復元できる。
4. **外部データを安全に参照する**: Google系（Gmail / Calendar / Drive）と Microsoft 系を公式コネクタで都度参照する。ローカルに同期コピーを作らない。
5. **秘書として日常業務を助ける**: 「今日やること」「予定とTODOの突き合わせ」などを、根拠（サービス名・リンク・日付）つきで返す。
6. **開発もできる**: 開発依頼を、agentic-harness を平易化した「やさしいハーネス」に接続する。
7. **報告が対象読者に伝わる**: ユーザー向け文言は「わかりやすい標準語彙」（一般的な技術用語はそのまま使い、馴染みの薄い語だけ初出で簡潔に補足。幼稚なメタファーは使わない）、報告は「3行以内・次に何が起きるかを一言」（`docs/spec/ui.md` 準拠）。

## 非ゴール（作らないもの）

- **外部データのローカル同期層**（my-vault の `10_sources` 型）は作らない。GitHub Actions にも依存しない。
- **cc-company の部署制**（company/部署フォルダ・キーワード振り分け表・部署間 inbox 通知）は採用しない。メタファーは「秘書＋道具箱」。
- **`experience/case-NNN` の必須生成・`patterns/` の自動統合**は廃止。振り返り・パターン化は「振り返りして」で起動するオンデマンド動作にする。
- **push の自動化**はしない。自動コミットはローカルのみ。push はユーザーの明示指示時だけ。
- **国内チャット（Chatwork / LINE 等）**は初期スコープ外（公式リモート MCP がなく自作ラッパーが必要なため見送り）。
- Web アプリ・サーバー・独自 UI は作らない。成果物はマークダウンのスキル群・テンプレート・ルールファイル。

## 成功状態（この製品が「できた」と言える状態）

- 非エンジニアが説明を見ながら3コマンドでプラグインを入れられる。
- `/secretary` を初めて呼ぶと、やさしい数問だけで `secretary/`（AGENTS.md / inbox / docs / projects / memory）が生成され、`git init` まで完了している。
- 秘書が「今日やること」を、外部データの根拠つき・3行報告で返す。
- 記憶が空上書き・誤削除されず、`MEMORY.md` 索引が常に最新に保たれる。
- Google コネクタの接続を、手作業の Google Cloud Console 設定なしにガイドできる。
- 開発依頼が「やさしいハーネス」に接続され、計画→道具→確認→結果の進行が非エンジニアに見える形で走る。
- リポジトリが public / MIT で、元作者 Shin-sibainu/cc-company へのクレジット（MIT）を継承している。

## 未確定事項（Planner のデフォルト判断・要すり合わせ）

- **配布ホストの解釈**: DESIGN.md はエンドユーザー標準環境を「Claude デスクトップアプリ」とする一方、成果物は Claude Code プラグイン形式（marketplace.json / plugin.json / skills）。本 spec では **「Claude Code プラグイン形式で配布し、コネクタ接続は Claude 側の設定画面から行う」** ことを既定とする（第1回で Git/GitHub を習得している前提と整合）。異なる場合は要指摘。
- **検証方法**: 本製品はマークダウン資産であり実行アプリではないため、Evaluator の検証はライブ対話ではなくスクリプト化した検証を主とする。詳細は `docs/spec/rubric.md`。
